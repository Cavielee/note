Mysql Server 分为：

SQL 层 + Storage Engine 层



存储引擎是基于表的，每个表可以分别使用不同的存储引擎。

Mysql 表默认使用 `InnoDB` 存储引擎。

存储引擎存储数据，其实际上都是以文件的形式存储。Mysql会存储在 `data` 目录中，其中每一个数据库（Scheme）在 `data` 中会有一个目录。不同的表根据使用的存储引擎的存储模式创建相应的存储文件

初始化模块：启动Server时，初始化加载Mysql 的默认参数，如（xx.ini文件）。

连接管理模块：用于监听端口，启动socket监听。当请求到达时，会分发到连接进程模块

连接进程模块：类似于连接池，当一个请求来了，有空闲的连接则直接执行，没有则创建来执行。

用户模块：判断用户权限

命令分发器：如果该命令在查询缓存器中有缓存则直接返回缓存值，否则把执行命令分发给命令解析器（parser）解析。

查询缓存器：缓存结构类似于Map，查询语句为key，查询结果为value。（当数据变化（update、insert、delete）则缓存失效）



性能优化：

需求：例如查询相册中前三张照片的评论总数。

可以通过查询该相片对应的评论表的count(*)，但如果评论数很多则会耗时。

方案一：花费io是 1 + 3

1. 先查询前三张照片id
2. 根据获得的id轮询count(*)

方案二：花费io是 1 + 1

1. 先查询前三张照片id
2. 把评论进行 GROUP ，然后count(*)相应的id

方案三：花费io是 1

在照片表中，定义一个评论总数字段来记录，从而直接查询字段即可。
