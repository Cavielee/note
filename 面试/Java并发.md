## 1.多线程和单线程的区别和联系

线程是处理的最小执行单位。

并发：多个线程同时工作，由CPU进行上下文切换。提高了对客户端的响应，但是降低执行速度。

并行：由于出现了多核CPU，支持同时运行多个线程，因此使用多线程将大大增加对系统资源的利用率。



## 2.如何指定多个线程的执行顺序

1. 可以通过 Join() 方法，规定线程必须在 join 线程运行完后才能运行。
2. 通过逻辑判断，每个线程判断特定的标志才能往下执行。



## 3.线程和进程的区别

1. 进程是一个运行的程序，是系统资源分配和调度的单位。
2. 一个进程可以有多个线程，线程是进程中最小的执行单位。
3. 线程可以共享进程的共享资源和空间地址。
4. 线程上下文切换时，只需记录当前运行到哪一行代码，进程则需要保存当前CPU环境，和新被调度运行进程的CPU环境设置。



## 4. 多线程产生死锁的 4 个必要条件

1. 资源不能被剥夺，分配的资源在未使用完成前不能前行被剥夺
2. 互斥条件，一个资源只能被一个线程使用
3. 请求和等待条件，线程请求其他资源而阻塞时，对持有的资源不放
4. 循环等待，若干个线程互相阻塞，并等待对方释放持有的资源，从而造成环状。

## 6.sleep( n) 和 wait( n)、wait( ) 的区别

sleep(n)：该方法是 Thread 类的方法，指定线程进行睡眠，并进入阻塞状态；释放CPU，不释放锁资；当到达n秒后，会解除阻塞状态重新进入可运行状态，并等待 CPU 时间片。

wait(n)：该方法是 Object 类的方法，需要在 Synchronized 中使用，指定线程进入该锁资源的等待队列中，并进入阻塞状态；释放CPU，释放锁资源；当到达n秒后或被其他线程notify/notifyAll后，才会解除阻塞状态，并重新争取锁资源后才能进入可运行状态。



## 7.synchronized 关键字



synchronized 是一种互斥锁，可以修饰方法和代码块

修饰方法则默认是全局锁（.class），修饰代码块可以自定义为局部锁（对象）或全局锁

使用锁可以确保运行锁内代码时会以同步的形式执行（同一时间只有一个线程执行）

建议修饰代码块，把锁的范围控制在需要同步的代码

在 JDK1.8后，对 Synchronized 进行了一系列优化：例如自旋锁（通过表记录锁的等待时间，如果较小，可以循环获取锁）、锁的粗细优化、偏向锁等

底层实际是获取对象的moniter，如果对象 monitor 计数器为0则代表没有线程获取锁，进入（monitorenter）则获取 monitor，并在 monitor 的计数器+1，释放锁则(monitorexit)计数器-1。

可支持重入形式，会记录当前 monitor 的拥有者，每重入一次计数器相应的+1。



## 8.volatile 关键字

volatile 实现可视性和禁止重排序

可视性：被 volatile 修饰的成员变量，会使每个线程的工作缓冲区中的该变量值失效，每一次修改都会直接写入到主内存中，每一次读取都会从主内存中读取，因此确保了每次读取都一定是最新的值。

禁止重排序：由于系统会对指令进行优化，只要不是影响结果的指令，可以进行重排序，从而避免等待耗时的指令。但在多线程情况下可能导致其他线程读取到未初始化完成的值，从而导致出现问题。用 volatile 修饰的变量会在该变量的指令加上内存屏障，确保该指令的执行在后面指令执行的前面。



## 9.ThreadLocal（线程局部变量）关键字

由于共享变量在多线程中存在不安全，例如 Connection，每个线程的 Connection 互不干扰，但可能存在操作的过程中有线程把 Connection 关闭，导致其他线程错误。因此我们把 Connection 这个共享对象保存在 ThreadLocal 对象，每个使用该 ThreadLocal 的线程实际是把使用该对象的副本，并把该副本保存在各自线程中，各自线程对该副本的修改不会相互影响，也不会影响 ThreadLocal 的对象。



## 10.**Atomic** 关键字

原子性，常见有 JUC 库下的 Atomic 原子类，这些类提供了原子性的操作。

原子性：确保该段代码要不全部正确执行成功，要不全部执行失败回退。

主要采用 CAS + volatile 实现原子性，通过 volatile 获取最新的值，通过 CAS 的乐观锁形式，尝试去写入数据，如果旧的值与内存的值相同，则确保操作过程没有其他线程干扰，就会修改值；不相同则重试。



## 11.线程池有了解吗

常用的线程池

FixedThreadPool：每一个任务开启一个线程，有最大的线程数

CacheThreadPool：不限线程个数，空闲线程可以被自动回收

SingleThreadPool：只有单个线程的线程池

ScheduleThreadPool：可以设置超时，定期执行任务

ForkJoinPool：可以返回结果，且结果可以实现非阻塞

WorkStealingPool：使用双向队列，其他空闲线程可从正在执行任务的线程中偷取工作处理
